## Purpose

Enhance the existing SDD workflow with three capabilities:
1. **Codebase indexing** before generating specs (context-aware planning)
2. **AGENTS.md export** for tool-agnostic task execution
3. **Precision-guided navigation** with line numbers to reduce token usage

---

## Enhancement 1: Codebase Indexing

Before generating any SDD artifact (proposal, spec, design, tasks), build a codebase index:

### Indexing Protocol

```
1. Map project structure (depth 2):
   - rg --files | head -500 | sort

2. Identify key architectural files:
   - Entry points (index.ts, main.go, app.py)
   - Config files (tsconfig, package.json, Cargo.toml)
   - Type definitions (types/, interfaces/, models/)
   - Test infrastructure (vitest.config, jest.config, conftest.py)

3. Extract public API surface:
   - Exported functions/classes per module
   - Route definitions
   - CLI command registry

4. Build dependency summary:
   - Direct dependencies (package.json, go.mod, requirements.txt)
   - Internal module graph (top-level imports only)
```

### Index Output Format

```
## Codebase Index: [project-name]

### Structure (247 files, 12 directories)
src/
  auth/       — JWT middleware, token refresh, RBAC
  routes/     — 8 API route files, Express router
  services/   — 5 business logic modules
  types/      — shared TypeScript interfaces
tests/        — vitest, 85% coverage

### Public API (23 exports)
- authMiddleware(req, res, next) — src/auth/middleware.ts:15
- createUser(data: UserInput) — src/services/user.ts:42
- [...]

### Dependencies (18 direct)
- express@4.18, jsonwebtoken@9.0, prisma@5.x
- vitest@4.1 (dev), supertest@6.3 (dev)
```

This index is included as context for ALL subsequent SDD artifacts.

## Enhancement 2: AGENTS.md Export

After generating SDD tasks, export them as tool-agnostic AGENTS.md:

```markdown
# AGENTS.md — Generated by SDD Pipeline

## Context
[Condensed codebase index]

## Tasks

### Task 1: [title]
**Files**: src/auth/jwt.ts, src/types/auth.ts
**What**: [description]
**Acceptance**: [criteria]
**Dependencies**: none

### Task 2: [title]
**Files**: src/routes/api.ts
**What**: [description]
**Acceptance**: [criteria]
**Dependencies**: Task 1
```

This format works with Claude Code, Codex, Gemini CLI, and Cursor.

## Enhancement 3: Precision-Guided Navigation

Replace full file content with line-number references:

### Instead of This (500 tokens):
```
The authMiddleware function currently validates tokens like this:
[entire function body pasted]
We need to change the expiry check on line 42.
```

### Do This (50 tokens):
```
Change src/auth/middleware.ts:42 — the expiry check in authMiddleware.
Current: `if (decoded.exp < Date.now())` 
Should be: `if (decoded.exp < Math.floor(Date.now() / 1000))`
```

### Navigation Rules

1. **Always include file:line** — `src/auth/jwt.ts:42`
2. **Quote only the relevant line(s)** — not the whole function
3. **Include 1 line of context** when the change is ambiguous
4. **Reference by symbol name** when line numbers may shift: `authMiddleware() in src/auth/middleware.ts`

### Token Savings

| Pattern | Before | After | Savings |
|---------|--------|-------|---------|
| Full function context | 500 tokens | 50 tokens | 90% |
| Full file for 1 change | 2000 tokens | 80 tokens | 96% |
| Multi-file spec | 8000 tokens | 400 tokens | 95% |

## Integration with Existing SDD Commands

This skill enhances, not replaces, the existing SDD commands:

| Command | Enhancement |
|---------|------------|
| `/sdd:new` | Runs codebase indexing first |
| `/sdd:explore` | Uses index for context-aware exploration |
| `/sdd:spec` | Includes precision references instead of full code |
| `/sdd:tasks` | Adds AGENTS.md export step |
| `/sdd:apply` | Uses line-number navigation for changes |
| `/sdd:verify` | References spec line numbers for traceability |

---

## Rules

1. **Always index before planning** — 5 seconds of indexing saves minutes of context
2. **Export AGENTS.md after tasks** — makes execution tool-agnostic
3. **Use file:line references** — never paste full files into specs
4. **Keep the index fresh** — re-index when the codebase changes significantly
5. **Index is L1 detail level** — not L2 (full content), not L0 (just filenames)
