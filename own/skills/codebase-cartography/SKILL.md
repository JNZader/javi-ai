# Codebase Cartography

Generate structured `codemap.md` files that give AI assistants instant architectural context. Reduces hallucinated file paths, wasted token budget, and repeated codebase exploration.

---

## Core Principle

AI agents perform dramatically better when they understand a codebase's shape before diving into files. A `codemap.md` captures: what the project is, how it's organized, where important code lives, and what it depends on.

**Token Budget Rule**: Under 2000 tokens (~1500 words). For monorepos, generate per-package codemaps.

---

## Codemap Output Format

```markdown
# Codebase Map

> Auto-generated by codebase-cartography. Last updated: {DATE}

## Project Overview
- **Name**: {name}
- **Language**: {language}
- **Framework**: {framework}
- **Package Manager**: {pm}
- **Monorepo**: {yes/no}
- **Test Framework**: {test}

## Architecture
**Pattern**: {detected pattern}

## Directory Structure
{tree output, max 80 files, depth based on project size}

## Key Files
| File | Role |
|------|------|
| {entry point} | {description} |

## Dependencies (Key)
- **{dep}** {version} — {role}

## Entry Points
- `{file}` — {description}

## Patterns Detected
- {architectural patterns found}
```

---

## Generation Steps

1. Detect language, framework, package manager, monorepo status
2. Detect architecture pattern (hexagonal, feature-based, MVC, layered, Go standard)
3. Generate filtered directory tree (exclude build artifacts, node_modules, .git)
4. Find entry points (main files, config files, Docker)
5. Extract top 10 dependencies
6. Write codemap.md under 2000 tokens

---

## Placement & Integration

Place `codemap.md` in project root. AI tools auto-load it:

| Tool | Mechanism |
|------|-----------|
| Claude Code | Reference in CLAUDE.md |
| OpenCode | Reference in agent instructions |
| Cursor | Include in .cursorrules |
| Aider | `aider --read codemap.md` |

**Token savings**: Codemap costs ~500-2000 tokens to load but saves 5000-20000 tokens of exploration.

### Refresh Triggers

Regenerate when: new module added, framework migration, major dependency change, quarterly review.
Do NOT regenerate for: new files in existing modules, version bumps, bug fixes, adding tests.

---

## Anti-Patterns

1. **Including file contents** — Codemap shows structure, not code. AI reads code separately.
2. **Listing every file** — Use depth limits and annotations for large directories.
3. **Hardcoding absolute paths** — Always use relative paths.
4. **Stale codemaps** — Worse than no codemap. Add generation date, refresh quarterly.
5. **Over-annotating** — Terse annotations only. This is orientation, not documentation.
6. **Mixing concerns** — No runtime behavior, business logic, or deployment topology. Structure only.
